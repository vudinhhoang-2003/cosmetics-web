Luồng xác thực JWT
CHƯƠNG 4 — THIẾT KẾ
// Sequence Diagram: Login Flow Client Backend Database | | | |-- POST /auth/login -→| | | {email, password} | | | |-- SELECT user ----→| | |←-- user record ----| | | | | | verify bcrypt | | | generate JWT | | | | |←-- 200 OK ----------| | | {access_token, | | | refresh_token} | | | | | | | | |-- API Request ------→| | | Bearer {token} | | | | verify signature | | | check expiry | |←-- Response ---------| | | | | |-- 401 Expired -----→| | |-- POST /refresh ---→| | |←-- New access token -| |
🔐 Chi tiết bảo mật
  • Access Token: 30 phút (HS256)
  • Refresh Token: 7 ngày
  • Bcrypt cost 12 – chống brute force
  • HTTPBearer scheme chuẩn
  • Token signature không thể giả mạo
  • Auto refresh khi 401 (axios interceptor)
  • require_admin dependency cho admin endpoints
